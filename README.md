# 🎓 Мой Личный Тренажер Тестов (Сайт-Приложение)

Красивый, современный, полностью русский и автономный сайт-тренажер для изучения теории и сдачи тестов. Проект разработан как одностраничное веб-приложение (SPA) с использованием **HTML5, Tailwind CSS и чистого JavaScript (ES6)**. Для работы не требуется серверная часть, базы данных или сборка — сайт полностью автономен и работает прямо в вашем браузере!

---

## ✨ Основные возможности

1. **📚 Учебник**:
   - Список всех вопросов из базы с подсветкой исключительно правильных ответов.
   - Полнотекстовый поиск по ключевым словам в вопросах и вариантах.
   - Удобные фильтры: "Все", "С одним верным ответом" и "С несколькими вариантами ответов".
2. **📝 Режим «Экзамен»**:
   - Настройка сессии: выбор количества вопросов (5, 10, 20, 50 или все сразу) и порядка вопросов (случайный или по порядку добавления).
   - Интерактивный интерактивный интерфейс: подсветка выбранных вариантов, проверка в реальном времени, таймер времени и детальные результаты.
   - Поддержка нескольких правильных ответов в одном вопросе.
3. **❌ Работа над ошибками (Персональный тренер)**:
   - Все неверно решенные вопросы на экзамене автоматически заносятся в этот раздел.
   - Выводится подробное сопоставление: вопросы с вариантами ответов, подсветка правильных вариантов и того, что именно выбрал пользователь в момент совершения ошибки.
   - **Умное авто-очищение**: если вы перепройдете тест и ответите на вопрос из ошибок правильно, он автоматически исчезнет из этого списка!
   - Возможность запустить экзамен исключительно по вашему списку ошибок.
4. **⚙️ Интерактивная база тестов и ИИ-помощник**:
   - Импорт `.json` файлов или вставка сырого текста JSON прямо в браузере. Тесты вступают в силу мгновенно и сохраняются в `localStorage` (не пропадут при обновлении страницы).
   - Экспорт текущей базы вопросов в `.json` файл для резервного копирования или редактирования.
   - Встроенный копируемый **Промт для ИИ**, который преобразует любые ваши сырые тесты в нужный формат за одну секунду!

## 🚀 Как запустить сайт локально

Поскольку сайт является полностью статическим, у вас есть два простых способа его запуска:

### Способ 1: Прямой запуск (Два клика)
Просто откройте файл `index.html` в любом современном веб-браузере (Google Chrome, Firefox, Safari, Edge, Яндекс.Браузер), дважды кликнув по нему на своем компьютере.
*Примечание: Если вы запускаете сайт напрямую как файл, браузеры могут блокировать чтение внешнего файла `tests.json` из-за ограничений безопасности (CORS). В этом случае сайт автоматически переключится на встроенную резервную копию базы данных. Чтобы загрузить свои тесты, вы можете просто импортировать их через панель управления на самом сайте.*

### Способ 2: Запуск через локальный веб-сервер (Рекомендуется)
Если у вас установлен Python, вы можете запустить быстрый веб-сервер в корневой папке проекта:
```bash
python3 -m http.server 8000
```
После этого откройте в браузере адрес: `http://localhost:8000`. В этом режиме локальный файл `tests.json` будет считываться безупречно!

## 🛠️ Как добавить свои тесты

Вы можете добавить новые вопросы двумя способами:

### Способ A: Постоянное добавление в коде (Рекомендуется)
1. Откройте файл `tests.json` в корневом каталоге проекта в любом текстовом редакторе.
2. Добавьте свои вопросы, строго соблюдая формат JSON, описанный ниже.
3. Сохраните файл и обновите страницу сайта.

### Способ Б: Динамический импорт через сайт (Без изменения файлов)
1. Перейдите во вкладку **«База тестов и ИИ»**.
2. В блоке справа выберите один из вариантов:
   - Нажмите **«Выбрать JSON файл»** и загрузите ваш файл `.json` с компьютера.
   - Или вставьте сырой текст JSON в текстовое поле и нажмите **«Загрузить из текста»**.
3. Новые тесты мгновенно заменят стандартные в текущей сессии вашего браузера. Вы можете экспортировать их обратно в любой момент, нажав **«Скачать текущие тесты (JSON)»**.

## 📐 Формат JSON для тестов

Любой файл тестов должен представлять собой массив объектов следующей структуры:

```json
[
  {
    "id": 1,
    "question": "Какие планеты Солнечной системы относятся к газовым гигантам?",
    "options": [
      "Марс",
      "Юпитер",
      "Сатурн",
      "Венера"
    ],
    "correct": [1, 2]
  },
  {
    "id": 2,
    "question": "Какое животное считается самым крупным млекопитающим на Земле?",
    "options": [
      "Африканский слон",
      "Синий кит",
      "Гигантский кальмар",
      "Кашалот"
    ],
    "correct": [1]
  }
]
```

### 📝 Спецификация полей:
- `id` — уникальный порядковый номер вопроса (начиная с 1).
- `question` — текст вопроса в виде строки.
- `options` — массив строк со всеми вариантами ответов.
- `correct` — массив чисел, содержащий **индексы правильных вариантов** из массива `options` (индексация начинается с **0**).
  - *Пример:* Если правильные варианты — "Юпитер" (2-й по счету) и "Сатурн" (3-й по счету), их индексы в массиве будут `[1, 2]`.
  - *Важно:* Даже если правильный ответ всего один, его индекс всё равно указывается внутри массива, например `[1]`.

## 🤖 Промт для ИИ (Преобразование тестов в JSON)

Чтобы вам не пришлось вручную переводить ваши тесты в формат JSON, скопируйте следующий промт и отправьте его в ChatGPT, Claude или любой другой ИИ вместе с вашими текстовыми тестами:

```text

Формат JSON должен быть строго массивом объектов, где каждый объект содержит:
- "id": уникальный числовой идентификатор (начиная с 1 по порядку)
- "question": строка с текстом вопроса
- "options": массив строк со всеми возможными вариантами ответов
- "correct": массив чисел (0-индексированные номера правильных вариантов ответов). То есть, первый вариант — это 0, второй — 1, и так далее. Если правильных ответов несколько, укажи все их индексы в массиве.

ПРАВИЛА:
1. Сверяй индексы правильных ответов очень внимательно! Не перепутай 1-индексацию с 0-индексацией.
2. Не изменяй оригинальные формулировки вопросов и вариантов ответов.
3. Твой ответ должен состоять ТОЛЬКО из валидного кода JSON внутри одного блока разметки ```json ... ```. Не добавляй никаких вступительных или заключительных слов, объяснений или комментариев.

1. Какая столица у Франции?

- Синий

Правильные ответы: Зеленый, Белый, Красный
```

## 📂 Структура файлов проекта
- `index.html` — главная веб-страница (SPA-разметка, сайдбар, вкладки, документация).
- `app.js` — логика приложения, таймер, загрузка/сохранение вопросов, результаты экзаменов, работа над ошибками.
- `styles.css` — стили, плавные анимации появления и кастомный дизайн скроллбаров.
- `tests.json` — основная база тестов по умолчанию (можно редактировать).
