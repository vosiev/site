Документация: Формат тестов и Как пользоваться

Этот сайт полностью автономен и работает прямо в вашем браузере. Вы можете использовать два способа добавления тестов в систему:

1

Постоянное сохранение в коде (Рекомендуется)

Откройте корневую папку репозитория и найдите файл tests.json. Отредактируйте его или замените своим в таком же формате. Любой запуск сайта будет автоматически подтягивать ваши вопросы!

2

Быстрый импорт через браузер (Без кодинга)

Вы можете загрузить готовый JSON-файл или просто вставить сырой JSON-текст прямо на этой панели справа! Вопросы мгновенно вступят в силу и сохранятся в памяти браузера (localStorage) — они останутся даже при перезагрузке!

Технический формат JSON вопросов:

База вопросов — это массив JSON-объектов. Каждый объект должен соответствовать следующей схеме:

[
  {
    "id": 1,
    "question": "Текст вопроса с ОДНИМ правильным ответом?",
    "options": [
      "Вариант 1 (Индекс 0)",
      "Вариант 2 (Индекс 1) - ПРАВИЛЬНЫЙ",
      "Вариант 3 (Индекс 2)"
    ],
    "correct": [1] 
  },
  {
    "id": 2,
    "question": "Текст вопроса с НЕСКОЛЬКИМИ правильными ответами?",
    "options": [
      "Правильный вариант A (Индекс 0)",
      "Неправильный вариант B (Индекс 1)",
      "Правильный вариант C (Индекс 2)"
    ],
    "correct": [0, 2]
  }
]
  • id: Уникальное число (ID вопроса)
  • question: Текст вашего вопроса
  • options: Список вариантов ответов (строковый массив)
  • correct: Массив индексов верных вариантов, начиная с 0. Если правильный ответ один — всё равно передается массивом из одного числа (например [1]).
ИИ-Помощник

Универсальный Промт для ChatGPT / Claude

Скопируйте этот промт, вставьте его в любую нейросеть (ChatGPT, Claude, Gemini) и отправьте вместе со своими текстовыми тестами. ИИ мгновенно вернет готовый код JSON, который можно сразу загрузить справа или вставить в tests.json!

Ты — опытный ИИ-ассистент, специализирующийся на структурировании данных. Твоя задача — преобразовать предоставленный список вопросов и вариантов ответов в правильный формат JSON для личного тренажера тестов.

Формат JSON должен быть строго массивом объектов, где каждый объект содержит:
- "id": уникальный числовой идентификатор (начиная с 1 по порядку)
- "question": строка с текстом вопроса
- "options": массив строк со всеми возможными вариантами ответов
- "correct": массив чисел (0-индексированные номера правильных вариантов ответов). То есть, первый вариант — это 0, второй — 1, и так далее. Если правильных ответов несколько, укажи все их индексы в массиве.

ПРАВИЛА:
1. Сверяй индексы правильных ответов очень внимательно! Не перепутай 1-индексацию с 0-индексацией.
2. Не изменяй оригинальные формулировки вопросов и вариантов ответов.
3. Твой ответ должен состоять ТОЛЬКО из валидного кода JSON внутри одного блока разметки ```json ... ```. Не добавляй никаких вступительных или заключительных слов, объяснений или комментариев.

Пример исходного текста тестов:
1. Какая столица у Франции?
а) Лондон
б) Берлин
в) Париж
г) Рим
Правильный ответ: в

2. Какие цвета есть на флаге Италии?
- Синий
- Зеленый
- Белый
- Красный
Правильные ответы: Зеленый, Белый, Красный

Пример твоего ответа (JSON):
[
  {
    "id": 1,
    "question": "Какая столица у Франции?",
    "options": ["Лондон", "Берлин", "Париж", "Рим"],
    "correct": [2]
  },
  {
    "id": 2,
    "question": "Какие цвета есть на флаге Италии?",
    "options": ["Синий", "Зеленый", "Белый", "Красный"],
    "correct": [1, 2, 3]
  }
]
